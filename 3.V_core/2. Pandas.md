# Декомпозиция пошагового сита фильтрации

**Дата фиксации:** 24.07.2026

**Целевая задача:** Построение отказоустойчивого промышленного конвейера очистки данных. Запечатывание физики пошавого выделения логических масок-клише в RAM компьютера и защита от скрытых капканов входящего типа данных.

## ЦЕХ №1: ИНИЦИАЛИЗАЦИЯ И СБОРКА КАРКАСА (DIL И DAFLIST)

Включаем в работу данного сеанса библиотеку Pandas и создаем промышленную двухмерную матрицу на базе структуры DIL (Dict in List).

### Эталонный код инициализации:

# Подключаем к данной сессии библиотеку pandas, придавая короткое имя для скоростного вызова
import pandas as pd

# Создаем скелет будущей таблицы. Данная синтаксическая конструкция является фундаментом 
# по созданию таблиц в Python, а именно словари в списке, иначе — конструкция DIL (Dictionary in List).
# По правилу "все через переменную", адрес сырых данных закладываем в переменную.
raw_data = [
    {"Объект": "ЖК центральный", "Поставка_тонн": 45, "Брак": False},
    {"Объект": "Объект Восточный", "Поставка_тонн": 38, "Брак": True},
    {"Объект": "Участок Северный", "Поставка_тонн": 50, "Брак": False}
]

# Чтобы из заготовки таблицы реализовать саму таблицу, используется инструмент DataFrame из библиотеки Pandas.
# Используется личная аббревиатура-напоминалка — DAFLIST (DataFrame from List).
df = pd.DataFrame(raw_data)

## ЦЕХ №2: ПОШАГОВЫЕ ОПЕРАТОРЫ ИЗОЛЯЦИИ И СЛОЖНЫЕ КЛИШЕ

Разбираем физику тактического разделения инлайновой фильтрации на автономные рубежи контроля в оперативной памяти компьютера.

### Код декомпозиции пошаговых масок:

# ТАКТИЧЕСКИЙ МАНЕВР: Разбиваем инлайновую "китайскую стену" на изолированные шаги и переменные.
# Создаем клише из нескольких условий. В отличие от базового Python, здесь мы работаем векторами,
# поэтому вместо слов "and/or" применяем побитовые операторы. Знак "|" означает ИЛИ.

# Шаг А: Создаем изолированное клише условий №1 (Фильтр по дефициту тоннажа бетона)
mask_tons = df["Поставка_тонн"] < 45

# Шаг Б: Создаем изолированное клише условий №2 (Фильтр по маркеру брака из лаборатории)
mask_defect = df["Брак"] == True

# Шаг В: Склеиваем два независимых трафарета через побитовый оператор ИЛИ (|) в одно монолитное клише.
# КРИТИЧЕСКИЙ СИНТАКСИС: каждое условие (маска) ОБЯЗАНО быть изолировано круглыми скобками ().
# Иначе процессор запутается в приоритетах операций и обрушит конвейер с аварией ValueError.
complex_clishe = (mask_tons) | (mask_defect)

print("Как выглядит сложное клише в RAM:")
print(complex_clishe)

# Накладываем клише (маску) на таблицу, тем самым скрывая лишнее. 
# Исходная таблица не меняется, новая таблица с данными, сохранившимися после применения маски, 
# физически формируется в RAM по правилу "Все действия через новую переменную" (Закон P).
df_problematic = df[complex_clishe]

print("\nТаблица проблемных поставок (комбинированный Where)")
print(df_problematic)

## ALL_CODE

import pandas as pd

raw_data = [
    {"Объект": "ЖК центральный", "Поставка_тонн": 45, "Брак": False},
    {"Объект": "Объект Восточный", "Поставка_тонн": 38, "Брак": True},
    {"Объект": "Участок Северный", "Поставка_тонн": 50, "Брак": False}
]
df = pd.DataFrame(raw_data)

# Создаем клише из нескольких условий через промежуточные такты RAM
mask_tons = df["Поставка_тонн"] < 45
mask_defect = df["Bрак"] == True
complex_clishe = (mask_tons) | (mask_defect)

print("Как выглядит сложное клише в RAM:")
print(complex_clishe)

# Накладываем маску на исходное хранилище по закону пошаговой изоляции данных
df_problematic = df[complex_clishe]

print("\nТаблица проблемных поставок (комбинированный Where)")
print(df_problematic)
