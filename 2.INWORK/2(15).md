## ШАГ 14: Динамические массивы (List) и индексация (Физика адресов в RAM)

```python
# Объявляем список объектов (индексация с 0)
object_list = ["ЖК Центральный", "Объект Восточный", "Участок Северный"]

# Вызов элементов: прямой и реверс (-1)
first_object = object_list[0]
print("Первый запуск", object_list[0])
print("Последний рубеж", object_list[-1])

# Подсчет количества элементов
print("Всего объектов в ленте", len(object_list))
```

## АНАТОМИЯ ПРОЦЕССА: (Физика динамических массивов и индексов)

1.  **Вводные переменные:** Создание списка `object_list` выделяет в RAM последовательную ленту адресов-ссылок.
2.  **Закон нулевого рубежа:** Индексация начинается с 0. Скобки `[]` — это прямое указание процессору: «Зайди, найди, выведи».
3.  **Реверс через -1:** Хитрый маневр, позволяющий получить последний элемент, не зная точной длины.
4.  **Функция-счетчик len():** Инструмент, пересчитывающий физические отсеки, аналог SQL-запроса `COUNT(*)`.

## ЛОВУШКА ДВОЙНОЙ ИНДЕКСАЦИИ (АНАТОМИЯ ОШИБКИ):
Ошибка `first_object[0]` выводит первую букву строки, а не элемент списка. Индексация — это инструмент и для списка (общего объекта), и для строки (символов внутри).

## СМЫСЛ ДЛЯ ПРОФЕССИИ:
Понимание работы с индексами — база для автоматизации ПТО (напр., работа с шифрами).
